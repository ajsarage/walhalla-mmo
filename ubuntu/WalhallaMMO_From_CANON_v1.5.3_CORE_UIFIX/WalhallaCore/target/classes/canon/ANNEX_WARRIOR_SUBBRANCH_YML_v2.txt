------------------------------------------------------------------
1. PROPÓSITO DEL ANNEX
------------------------------------------------------------------

Este documento describe de forma literal y no interpretativa
los archivos YML existentes asociados a las subramas del GUERRERO.

Estos YML NO definen la XP directamente, pero SÍ definen:
- tiers (niveles) de desbloqueo
- requisitos de nivel para desbloquear
- costes de desbloqueo (puntos/moneda)
- modo corrupto (estilo Warframe) como elemento de build

Por tanto, estos YML están conectados a progresión mediante:
- nivel de subrama / rama (required_level)
- puntos y costes (cost_skill, cost_wcraft)

------------------------------------------------------------------
3. ESQUEMA ESTRUCTURAL OBSERVADO (TOP LEVEL)
------------------------------------------------------------------

Cada archivo de subrama tiene estas claves top-level:

- id            (string)
- name          (string)
- type          (string)  -> valor observado: "warrior"
- levels        (map/dict)
- corrupto      (map/dict)

------------------------------------------------------------------
4. BLOQUE "levels" (TIERS DE DESBLOQUEO)
------------------------------------------------------------------

"levels" es un mapa (dict) que contiene, de forma consistente:

- tier1
- tier2
- tier3
- tier4
- tier5

Cada "tierX" observado contiene exactamente:

- id             (string)
- name           (string)
- icon           (string)
- description    (string)
- required_level (int)
- cost_wcraft    (int)
- cost_skill     (int)

Interpretación permitida (factual, por semántica de campos):
- required_level indica nivel mínimo requerido
- cost_skill indica coste en puntos (skill points / puntos de rama-subrama)
- cost_wcraft indica coste en moneda WCraft

Este ANNEX NO define cómo se obtiene cost_skill:
solo confirma que el YML exige ese coste para desbloquear.

------------------------------------------------------------------
5. BLOQUE "corrupto" (MODO CORRUPTO estilo Warframe)
------------------------------------------------------------------

"corrupto" existe en las tres subramas con estructura:

- id          (string)
- name        (string)
- icon        (string)
- description (string)

Observación factual:
- no aparecen campos de required_level ni costes dentro de "corrupto"
  en el estado actual de los YML auditados.

Por tanto, los requisitos/costes del modo corrupto, si existen,
deben estar:
- en otro documento
- o en otra capa de lógica/código
- o pendientes de ampliación del formato YML

------------------------------------------------------------------
6. CONEXIÓN CON XP / PROGRESIÓN (SIN CONTRADICCIÓN)
------------------------------------------------------------------

Estos YML:

- NO contienen XP numérica (no definen “cuánta XP ganas”).
- SÍ contienen requisitos y costes dependientes de progresión:
  - required_level
  - cost_skill
  - cost_wcraft

Por tanto:

- La XP y el nivel se gestionan en el sistema de progresión (Core).
- Los YML actúan como CATÁLOGO de desbloqueos y condiciones.
- El jugador construye su build decidiendo qué desbloquea
  según nivel y puntos acumulados.

Esto es consistente con:
- la intención de build personalizable
- el sistema de puntos de rama/subrama
- los documentos de progresión existentes
