1) REGLA GENERAL DE COMBATE
--------------------------
Regla CR-G1:
En roles de combate, el uso de un arma o equipo
(ataques, golpes, disparos, bloqueos) SOLO progresa
la afinidad del arma o equipo correspondiente.

El uso por sí mismo NO genera XP de:
- rama
- subrama
- jugador


2) EVENTO DE MUERTE
------------------
Regla CR-G2:
La progresión normal de rama, subrama(s) y jugador
se produce cuando un enemigo es derrotado mediante
un evento de muerte validado por el Core.

Este evento es la fuente principal de progresión
en combate.


3) GUERRERO Y CAZADOR
--------------------
Aplican estrictamente las reglas CR-G1 y CR-G2.

- Uso del arma → afinidad
- Muerte con el arma → XP de rama, subrama(s), jugador


4) TANQUE — EXCEPCIÓN CANÓNICA
-----------------------------
Regla CR-T1:
El Tanque puede progresar su rama y subramas mediante
eventos de bloqueo exitoso con escudo.

Características:
- XP reducida respecto a un evento de muerte
- Evento validado por el Core
- Filtrado anti-repetición para evitar abuso

Regla CR-T2:
Cuando el Tanque derrota a un enemigo utilizando
habilidades o ataques propios de su escuela,
se aplica XP normal de:
- rama
- subrama(s)
- jugador

El evento de muerte sigue siendo la fuente de
progresión más relevante.


5) PROGRESIÓN PARALELA
---------------------
Regla CR-G3:
Un mismo evento válido puede otorgar, de forma paralela:
- XP de rama
- XP de una o varias subramas
- XP de jugador
- XP de afinidad (según balance)

La asignación final es responsabilidad exclusiva del Core.


6) AUTORIDAD
------------
El Core es la única autoridad para:
- validar eventos de uso, bloqueo y muerte
- decidir qué XP se aplica
- filtrar abuso o repetición

La UI y los módulos externos no deciden progresión.
