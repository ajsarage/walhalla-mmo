SECCIÓN Z — COMBATE (Z6 / Z7)
============================================================

------------------------------------------------------------
Z6 — COMBATE BASE
------------------------------------------------------------

Estado: IMPLEMENTADO

Descripción:
- Existe un sistema explícito de ataque
- Los ataques PRIMARY y SECONDARY:
  - generan un contexto de ataque
  - solicitan validación a WalhallaCore
  - solo se ejecutan si el Core responde APPROVED

Responsabilidades:
- WalhallaCore:
  - valida ataques
  - decide reglas
  - mantiene autoridad total
- WalhallaCombat:
  - NO decide reglas
  - NO valida permisos
  - ejecuta únicamente tras aprobación del Core

Limitaciones actuales:
- No existe HUD de daño
- No existe feedback visual
- No existe balance visible para el jugador
- El sistema es técnico, no experiencial

------------------------------------------------------------
Z7 — WEAPONMODS
------------------------------------------------------------

Estado: IMPLEMENTADO (ARQUITECTURA) / TESTEO APLAZADO

Descripción:
- Integración correcta y canónica:
  WeaponMods → Combat → Core

Pipeline obligatorio (confirmado):
1) Core.validateAttack
2) WeaponMods modifican CombatContext
3) Core.resolveCombat

Comportamiento actual:
- WalhallaCombat:
  - NO calcula daño
  - NO interpreta WeaponMods
  - NO balancea
  - Actúa únicamente como integrador de contexto

- WeaponMods:
